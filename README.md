# 청춘캠퍼스

> 한줄 요약: 청춘캠퍼스 서비스의 전체 구조, 핵심 기능, 관리자 분리 원칙을 빠르게 이해하기 위한 프로젝트 개요 문서입니다.

청춘캠퍼스는 휴대폰 모바일 브라우저를 기본 실행 환경으로 하는 대학생 대상 소개팅 서비스 프로젝트입니다.

서비스는 두 가지 흐름으로 운영됩니다.
- `로테이션 소개팅`
- `1대1 소개팅`

## 서비스 개요
- 메인 페이지에서 `로테이션 소개팅`, `1대1 소개팅`, `후기`, `이용 안내`로 이동합니다.
- 로테이션 소개팅은 행사형 서비스입니다.
  남자 11명, 여자 11명을 모집해 2~3시간 동안 단체 소개팅을 진행합니다.
- 1대1 소개팅은 프로필과 선호 조건을 바탕으로 상대를 연결하는 매칭형 서비스입니다.
- 각 신청 페이지에서 입력받은 정보는 DB에 저장되고, 관리자 페이지에서 바로 확인할 수 있어야 합니다.
- 관리자 페이지에서는 받은 신청 정보를 `로테이션 소개팅 신청`과 `1대1 매칭 신청`으로 나누어 관리합니다.
- 1대1 매칭에서는 `선호 스타일 우선순위`를 기준으로 자동매칭을 실행하고, 필요 시 관리자가 수동매칭을 진행할 수 있습니다.

## 주요 페이지
- `index.html`
  메인 랜딩 페이지
- `rotation.html`
  로테이션 소개팅 소개 페이지
- `rotation-apply.html`
  로테이션 소개팅 신청 페이지
- `match.html`
  1대1 소개팅 소개 페이지
- `match-apply.html`
  1대1 소개팅 신청 페이지
- `reviews.html`
  후기 작성 및 후기 목록 페이지
- `guide.html`
  이용 안내 페이지
- `admin.html`
  관리자용 비공개 페이지
  메인 버튼과 공용 상단 메뉴에서는 노출하지 않음

## 데이터 흐름
- 사용자가 신청 페이지에서 정보를 입력합니다.
- 신청 데이터는 `로테이션 소개팅 신청` 또는 `1대1 소개팅 신청`으로 저장됩니다.
- `admin.html`에서 인증된 관리자가 신청 데이터를 조회합니다.
- 관리자 페이지는 `로테이션 관리`와 `1대1 매칭 관리`를 분리해서 보여줍니다.
- 1대1 매칭 관리에서는 선호 스타일 우선순위를 반영한 자동매칭과 관리자 수동매칭을 지원합니다.

## 상세 요구 조건
- 신청 페이지 제출 직후 데이터는 서비스 유형별로 저장되어야 하며, 관리자 페이지에서 새로고침 없이도 최신 목록으로 반영될 수 있는 구조를 목표로 합니다.
- 관리자 페이지 기본 화면은 `로테이션 신청 관리`와 `1대1 매칭 관리`를 명확히 분리해 혼선을 줄여야 합니다.
- 로테이션 신청 관리는 회차별 정원, 성별별 확정 인원, 대기열 순번, 취소 후 승급 여부를 확인할 수 있어야 합니다.
- 1대1 매칭 관리는 신청자 기본 정보, 본인 스타일, 선호 스타일 우선순위, 현재 상태, 매칭 후보 이력을 함께 보여줄 수 있어야 합니다.
- 자동매칭은 `선호 스타일 우선순위`를 최우선 기준으로 반영하고, 하드 조건 불일치 대상은 후보에서 제외해야 합니다.
- 수동매칭은 자동매칭 결과와 별개로 실행 가능해야 하며, 누가 왜 수동매칭을 했는지 운영 메모를 남겨야 합니다.

## 관리자 접근 정책
- `admin.html`은 일반 사용자 페이지와 분리된 별도 진입점으로 유지합니다.
- `index.html`, `rotation.html`, `rotation-apply.html`, `match.html`, `match-apply.html`, `reviews.html`, `guide.html`에서는 관리자 스크립트와 관리자 컴포넌트를 로드하지 않습니다.
- 프론트엔드 코드와 설정 파일에 관리자 비밀번호를 하드코딩하지 않습니다.
- 관리자 인증은 `커스텀 서버 세션` 또는 `Supabase Auth` 중 하나로 처리합니다.
- 관리자 전용 조회, 상태 변경, 자동매칭, 수동매칭은 서버 API 또는 Supabase Edge Functions를 통해 실행합니다.

## 서비스 구조

### 로테이션 소개팅
- 행사 소개
- 일정/장소 안내
- 회차 선택 신청
- 성별별 정원 관리
- 대기열 처리 가능
- 관리자 페이지에서 회차별 신청 목록과 확정/대기 상태를 관리

### 1대1 소개팅
- 서비스 소개
- 매칭 방식 안내
- 신청 폼 입력
- 프로필/선호 조건 저장
- 선호 스타일 우선순위 입력
- 자동매칭 후보 생성
- 관리자 수동매칭 및 승인 가능

## 사용자 안내
- 휴대폰에서 링크를 열어 사용하는 것을 기본으로 설계합니다.
- 메인 페이지에서 원하는 서비스를 선택합니다.
- 로테이션 소개팅은 행사 신청 흐름으로 이동합니다.
- 1대1 소개팅은 프로필 기반 신청 흐름으로 이동합니다.
- 후기 페이지에서는 후기를 남기고 다른 사용자의 후기를 확인할 수 있습니다.

## 관리자/운영 참고
- 신청 데이터는 개인정보를 포함하므로 일반 공개 대상이 아닙니다.
- 관리자 페이지는 `로테이션 신청 관리`와 `1대1 매칭 관리`를 분리해 보여주는 구조를 우선합니다.
- 로테이션 소개팅은 회차별 정원 관리와 참가 확정이 중요합니다.
- 1대1 소개팅은 신청 검토, 선호 스타일 우선순위 기반 자동매칭, 수동매칭, 상태 관리가 중요합니다.
- 후기 승인 정책을 적용할 경우 승인 화면은 `admin.html`에서만 제공합니다.
- 관리자 권한 검증은 서버 또는 Supabase RLS 정책으로 보장합니다.

## 기술 개요
- 프론트엔드: React 기반 UI 구성
- 백엔드/DB: Supabase Postgres
- 인증: 커스텀 서버 세션 또는 Supabase Auth
- 권한 제어: RLS + 서버 측 관리자 검증
- 배포: 정적 웹 배포 + 별도 관리자 진입점
- 기본 타겟 환경: 모바일 브라우저 우선, PC는 보조 지원
