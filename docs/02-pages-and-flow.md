# 내부 기획 02: 페이지 구조와 사용자 흐름

> 한줄 요약: 이 문서는 사용자 페이지와 관리자 페이지가 어떤 책임을 갖고 어떤 순서로 이동하는지 정의하는 화면 흐름 기준서입니다.

이 문서는 내부 기획용 문서입니다.
페이지 구성을 어떤 식으로 나눌지와 사용자의 이동 흐름을 정리합니다.

## 페이지 초안
- `index.html`
  메인 랜딩 페이지
- `rotation.html`
  로테이션 소개팅 소개 페이지
- `rotation-apply.html`
  로테이션 소개팅 신청 페이지
- `match.html`
  1대1 소개팅 소개 페이지
- `match-apply.html`
  1대1 소개팅 신청 페이지
- `reviews.html`
  후기 작성 + 후기 목록 페이지
- `guide.html`
  이용 안내 페이지
- `admin.html`
  관리자용 비공개 페이지
  메인 버튼과 공용 상단 메뉴에서는 노출하지 않음
  `로테이션 신청 관리` 섹션
  `1대1 매칭 관리` 섹션
  신청 목록 조회 및 상태 필터링
  신청자 상세 정보 확인
  자동매칭 실행 및 결과 확인
  수동매칭 실행
  매칭 상태 분류 및 완료 처리

## 페이지 로드 정책
- 공용 페이지는 사용자용 화면과 공용 컴포넌트만 로드
- 공용 페이지는 `src/pages/admin.js`, `src/components/admin-application-list.js`, `src/components/admin-rotation-panel.js`, `src/components/admin-match-panel.js`, `src/services/admin.js`를 로드하지 않음
- `admin.html`에서만 관리자 스크립트와 관리자 컴포넌트를 로드

## 기본 실행 환경
- 휴대폰 모바일 브라우저 우선
- PC는 보조 확인 환경

## 사용자 흐름

### 로테이션 소개팅
1. 메인 페이지 진입
2. 로테이션 소개팅 버튼 클릭
3. 소개 페이지 확인
4. 신청 페이지 이동
5. 회차 선택 및 신청
6. 신청 데이터 저장
7. 관리자 페이지의 `로테이션 신청 관리`에서 확인

### 1대1 소개팅
1. 메인 페이지 진입
2. 1대1 소개팅 버튼 클릭
3. 소개 페이지 확인
4. 신청 페이지 이동
5. 프로필, 선호 조건, 본인 스타일, 선호 스타일 우선순위 입력
6. 신청 데이터 저장
7. 관리자 페이지의 `1대1 매칭 관리`에서 확인
8. 선호 스타일 우선순위 기반 자동매칭 또는 수동매칭 진행

### 후기
1. 메인 페이지 진입
2. 후기 버튼 클릭
3. 후기 페이지 이동
4. 후기 작성 또는 후기 확인

### 관리자
1. `admin.html` 직접 진입
2. 커스텀 서버 세션 또는 Supabase Auth로 관리자 인증
3. `로테이션 신청 관리`와 `1대1 매칭 관리` 중 선택
4. 신청 목록과 상세 정보를 확인
5. 1대1 섹션에서 자동매칭 실행 또는 수동매칭 실행
6. 상태 변경 및 완료 처리

## 페이지별 상세 요구 조건
- `index.html`은 서비스 소개와 진입 버튼에 집중하고, 관리자 관련 진입 요소를 포함하지 않아야 합니다.
- `rotation-apply.html` 제출 완료 후에는 접수 상태, 대기열 가능성, 추후 안내 방식을 명확히 안내해야 합니다.
- `match-apply.html`에서는 스타일 우선순위를 단계형 UI로 입력받고, 중복 선택을 막아야 합니다.
- `admin.html` 진입 후 기본 화면은 최근 신청 현황 요약과 함께 `로테이션 신청 관리` 또는 `1대1 매칭 관리` 중 하나를 바로 선택할 수 있어야 합니다.
- `1대1 매칭 관리` 상세 화면에서는 `신청 정보`, `스타일 우선순위`, `자동매칭 후보`, `수동매칭 실행 영역`, `운영 메모`가 한 흐름으로 이어져야 합니다.
- 에러가 발생하면 공용 페이지는 사용자 친화적 메시지를 보여주고, 관리자 페이지는 원인 파악이 가능한 운영용 메시지를 추가로 보여줄 수 있어야 합니다.
