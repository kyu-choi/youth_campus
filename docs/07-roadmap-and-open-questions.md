# 내부 기획 07: 작업 순서와 미확정 항목

> 한줄 요약: 이 문서는 실제 구현 순서, 선행 작업, 아직 결정되지 않은 운영 기준을 추적하는 실행 계획 문서입니다.

이 문서는 내부 진행 순서 문서입니다.
무엇을 먼저 만들지와 아직 결정하지 않은 항목을 정리합니다.

## 작업 순서
1. 메인 버튼 구조를 `로테이션 소개팅 / 1대1 소개팅 / 후기 / 이용 안내` 기준으로 수정
2. 공용 페이지에서 관리자 스크립트와 관리자 컴포넌트 로드 제거
3. `admin.html` 비공개 관리자 페이지 유지
4. 로테이션 소개 페이지 생성
5. 로테이션 신청 페이지 생성
6. 1대1 소개 페이지 생성
7. 1대1 신청 페이지 생성
8. 신청 페이지 입력 데이터를 서비스별로 저장하는 구조 연결
9. 관리자 비밀번호 하드코딩 제거
10. 커스텀 서버 세션 또는 Supabase Auth 연결
11. 관리자 페이지에 `로테이션 신청 관리` 섹션 추가
12. 관리자 페이지에 `1대1 매칭 관리` 섹션 추가
13. 선호 스타일 우선순위 입력 구조 연결
14. 선호 스타일 우선순위 기반 자동매칭 로직 반영
15. 수동매칭 실행 및 매칭 완료 처리 기능 반영
16. 후기 페이지 생성
17. 모바일 브라우저 기준 최종 점검

## 아직 결정이 필요한 항목
- "로테이션 소개팅"과 "로케이션 소개팅" 중 최종 표현
- 메인 배지 문구 유지 여부
- 버튼 이미지를 사진형으로 할지 아이콘형으로 할지
- 로테이션 행사 회차 운영 방식
- 로테이션 정원 초과 시 대기열 기준
- 선호 스타일 우선순위를 몇 개까지 받을지
- 선호 스타일 우선순위 가중치를 어떻게 둘지
- 인증 구현에서 먼저 적용할 방식
  커스텀 서버 세션 또는 Supabase Auth
- 신청 상태값과 매칭 상태값 최종 확정
- 수동매칭 허용 기준
- 후기 공개를 자동으로 할지 관리자 승인 후 할지
- 매칭 완료 후 사용자 알림 발송 시점

## 다음 회차에 정하면 좋은 것
- 공통 프로필 필드 최종 확정
- 로테이션 전용 입력값 확정
- 1대1 전용 입력값 확정
- 자동매칭 하드 조건/소프트 조건 분리
- 스타일 우선순위 점수 표준 확정
- 관리자 화면 우선순위와 최소 기능 범위 정리
- 관리자 권한 모델
  서버 허용 목록 또는 `admin_roles`
- 모바일 기준 해상도와 테스트 범위 정리

## 단계별 완료 조건
- 신청 시스템 단계 완료 기준은 `사용자 제출 -> DB 저장 -> 관리자 페이지 조회`가 실제로 연결되는 것입니다.
- 로테이션 관리 단계 완료 기준은 회차별 신청 조회, 정원 반영, 대기열 처리, 상태 변경이 가능한 것입니다.
- 1대1 매칭 단계 완료 기준은 선호 스타일 우선순위 저장, 자동매칭 실행, 수동매칭 생성, 후보 승인까지 가능한 것입니다.
- 인증 단계 완료 기준은 비로그인 상태에서 관리자 데이터 접근이 차단되고, 로그인 후에만 관리자 액션이 가능한 것입니다.
- 문서 완료 기준은 각 기능의 입력값, 상태값, 처리 주체, 예외 처리 규칙이 다른 문서와 충돌 없이 정리된 상태입니다.
