# 내부 기획 04: 후기 시스템

> 한줄 요약: 이 문서는 후기 작성, 공개, 승인, 관리자 검수 흐름을 정의해 후기 기능이 운영 정책과 어긋나지 않도록 관리합니다.

이 문서는 내부 기획용 문서입니다.
후기 페이지에서 원하는 기능과 구현 순서를 정리합니다.

## 현재 원하는 기능
- 후기 작성 페이지 별도 구성
- 후기 작성 폼 제공
- 기존 후기 목록 조회
- 관리자 승인 후 공개 여부를 관리할 수 있는 구조 고려
- 휴대폰에서 읽기 쉽고 쓰기 쉬운 후기 UX 구성
- 로테이션 소개팅과 1대1 소개팅 이용자 모두 후기 작성 가능하도록 고려

## 운영 원칙
- 공개 후기 페이지는 사용자용 후기 UI만 로드
- 후기 승인 기능을 사용할 경우 승인 화면은 `admin.html`에서만 제공
- 관리자 페이지에서는 신청 관리 섹션과 후기 승인 섹션을 분리
- 승인 처리 권한은 서버 세션 또는 Supabase Auth로 인증된 관리자에게만 부여
- 프론트엔드에 관리자 비밀번호를 두지 않음

## 후기 페이지에서 받고 싶은 정보
- 닉네임
- 후기 제목
- 후기 내용
- 별점
- 참여 회차 또는 날짜
- 익명 여부
- 공개 동의 여부

## 후기 목록에서 보여주고 싶은 정보
- 닉네임
- 제목
- 내용
- 작성일
- 별점

## 작업 순서
1. 후기 페이지 레이아웃 구성
2. 후기 작성 폼 생성
3. 후기 카드 목록 UI 생성
4. 후기 데이터 저장/조회 연결
5. 공개 정책 반영
6. 관리자 승인 흐름이 필요하면 `admin.html`의 별도 섹션에 연결
7. 모바일 화면 기준 가독성 점검

## 상세 요구 조건
- 후기 작성은 실제 참가 또는 실제 매칭 완료 사용자만 가능하게 제한할지 여부를 정책으로 확정해야 합니다.
- 후기 공개 전에는 욕설, 개인정보, 연락처, 특정인 식별 정보 포함 여부를 검토할 수 있어야 합니다.
- 익명 후기는 운영 DB에는 작성자 식별 정보가 남더라도 사용자 화면에는 익명 기준으로만 노출해야 합니다.
- 관리자 반려 시 반려 사유를 남길 수 있어야 하며, 필요하면 재검토 상태를 둘 수 있습니다.
- 후기 공개 여부와 별개로, 관리자 페이지에서는 후기 원문과 승인 상태를 분리해서 볼 수 있어야 합니다.
